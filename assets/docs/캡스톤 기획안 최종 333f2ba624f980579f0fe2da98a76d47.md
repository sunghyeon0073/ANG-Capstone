## 01 — 프로젝트 기획내용

"작년에 누가 이 신청서 썼지? 어디 있지? 그냥 처음부터 다시 쓸까?"

과거 자료가 있다는 건 알지만, 찾는 데 시간이 더 걸리고 결국 매년 같은 문서를 새로 만들게 됩니다. 업무를 인수인계받은 신입은 맥락 없이 파일만 전달받고, 팀장은 누가 무슨 업무를 하고 있는지 따로 물어봐야 알 수 있습니다.

**이 문제는 대학 행정 부서, 학생회, 연구실까지 — 조직이 있는 곳이라면 어디서나 반복되는 구조적 비효율 적입니다.**

💡 *"이전에 한 일이 다른 사람에게, 그리고 다음에도 제대로 이어질 수 있다면 어떨까?"*
— 이 질문이 ANG의 출발점입니다.

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | ANG (AI Network Group) |
| 핵심 목표 | 반복 작업 감소 / 업무 흐름 시각화 / 조직 데이터 연속성 확보 |
| 1차 타깃 | 대학 행정 부서 ( 평생 교육원 등 )  |
| 차별점 | 쓸수록 조직 데이터가 자동 축적되는 구조 — 범용 협업툴과 다른 핵심 |

## 02 — 문제 정의

**조직이 반복적으로 겪는 4가지 비효율**

**문제 ①  매년 같은 문서를 처음부터 만든다**
행사 신청서, 예산안, 보고서 등 이전 파일을 찾기 어렵거나 공유가 안 되어 있어 탐색 시간 + 재작성 시간이 이중으로 낭비됩니다.

**문제 ②  팀 전체의 업무 현황을 파악하기 어렵다**
팀장은 각 팀원의 업무 진행 상황을 직접 물어봐야 합니다. 업무 편중이나 중복을 사전에 발견하기 어렵습니다.

**문제 ③  반복 일정인데  준비가 늦거나 잊는다**
매년 같은 시기에 반복되는 일정이지만, 전임자의 기록이 없거나 준비 시작 시점을 놓치게 됩니다.

**문제 ④  인수인계는 파일 전달로 끝난다**
가장 중요한 건 "왜 이렇게 했는지"인데, 지금의 인수인계는 파일 공유로 끝납니다. 맥락 없이 전달된 자료는 활용되지 않고 쌓이기만 합니다.

## 03 — 킬러 콘텐츠 (핵심 기능)

**3가지 기능이 유기적으로 연결됩니다**

**🗂 문서 재활용 시스템**

새 문서 작성 시, 제목이나 문서 유형을 입력 시, **유사한 과거 문서 리스트가 자동으로 나열**됩니다. 리스트에는 문서명·작성일이 함께 표시되며, **파일을 클릭하면 해당 문서의 내용을 미리보기로 확인**할 수 있습니다. 내용을 확인한 뒤 원하는 파일을 내려받은 후 업로드 시 해당 문서를 기반으로 초안이 자동 생성됩니다. PDF, DOCX, HWP(예정) 등 다양한 형식을 통합 지원합니다.

> 예시: 행사 지원 신청서 작성" 입력
→ 2023·2024·2025년 동일 행사 신청서 리스트 자동 나열
→ 2024년 파일 클릭 → 본문 내용 미리보기 확인 → 선택 → 날짜·담당자만 수정해서 완성
> 

**📅 업무 흐름 연동 캘린더**
단순 날짜 관리가 아닌, 과거 업무 패턴 기반으로 일정을 자동 제안합니다. 반복 업무는 다음 해 일정에도 자동 반영되어 준비 시작 시점을 절대 놓치지 않습니다.

> 예시: 매년 11월에 결산 업무 진행 → 10월 말에 "준비 시작" 알림 자동 발송 + 작년 결산 서류 리스트 함께 제공
> 

**🧩 부서 전용 모듈화**

> 처음부터 전체를 도입할 필요가 없습니다. 행정팀에는 공문·신청서 자동화 모듈, 학생회에는 예산 보고 모듈, 연구실에는 회의록 요약 모듈처럼 **부서의 업무 성격에 맞는 기능만 골라서 시작**할 수 있습니다. 필요한 것부터 붙이고, 쓸수록 데이터가 쌓여 점점 조직에 맞게 고도화됩니다.

## 04 — 기술 스택 및 구현 방식

> 🔒 ANG는 외부 API에 의존하지 않는 **로컬 AI 서버**로 구동됩니다. 학교 데이터가 외부로 나가지 않는 구조입니다.
> 

**LLM 엔진 — Ollama**
오픈소스 LLM을 로컬 서버에서 직접 실행. 인터넷 없이도 AI 동작. 사내 DB와 직접 연결해 부서 특화 답변 생성.
`Ollama` `LLaMA 3` `Mistral`

**문서 검색 — RAG**
Retrieval-Augmented Generation: 과거 문서를 벡터화해 저장. AI가 질문 시 관련 문서를 즉시 검색해 근거 기반 답변 생성.
`RAG` `LangChain` `ChromaDB`

**문서 디지털화 — OCR**
스캔 문서, 수기 신청서, PDF를 텍스트로 변환. AI가 즉시 읽고 요약·분류·초안 생성에 활용 가능.
`Python` `Tesseract` `PyMuPDF`

**업무 보드 — 웹 기반 UI**
칸반 보드, 캘린더, 문서 뷰어를 단일 웹 인터페이스로 제공. 별도 설치 없이 브라우저에서 바로 사용.
`React` `FastAPI` `PostgreSQL`

**배포 및 환경 관리 — Docker**
전체 서비스를 컨테이너로 패키징해 어떤 서버 환경에서도 동일하게 구동. 부서별 독립 환경 분리 및 빠른 배포·복구 지원.
`Docker` `Docker Compose`

## 05 — 경쟁 플랫폼 비교

**왜 기존 도구로는 부족한가**

노션, 슬랙, 구글 워크스페이스 등 기존 협업 툴은 공통적으로 **"지금 이 조직"의 데이터를 누적시키지 않습니다.**

| 비교 항목 | 기존 협업툴 (노션·슬랙 등) | 외부 AI (ChatGPT 등) | ANG |
| --- | --- | --- | --- |
| 문서 관리 | 저장 중심 | 텍스트 생성만 가능 | 재활용 중심 |
| 업무 관리 | 개인 할 일 위주 | 해당 없음 | 팀 전체 흐름 중심 |
| 일정 관리 | 직접 입력만 가능 | 해당 없음 | 과거 패턴 기반 자동 제안 |
| 인수인계 | 파일 전달로 종료 | 해당 없음 | 맥락 포함 업무 자산 전달 |
| 보안 | 외부 클라우드 저장 | 데이터 외부 전송 | 로컬 서버 완전 폐쇄 |
| 데이터 누적 | 사용자가 직접 관리 | 누적 불가 | 사용할수록 자동 축적 |

> ANG는 **"쓸수록 조직이 똑똑해지는"** 구조입니다. 범용 툴이 채우지 못하는 "조직 고유 데이터의 자산화"가 핵심 차별점입니다.

## 07 — 시장 타당성 (근거 자료)

**왜 지금, 왜 이 시장인가**

**📌 교육부 · 대학의 AI·디지털 전환을 정책적으로 지원 중**
교육부는 2025~2026년에 걸쳐 전문대학의 AI·디지털 전환(DX)을 체계적으로 지원하는 사업을 추진하고 있습니다. 이는 ANG의 도입 명분과 정확히 맞물립니다.
[출처: 교육부 공식 사업 공고](https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=294&boardSeq=105277&lev=0&m=020402&opType=N&page=1&s=moe)

**📌 KISA · 외부 AI 사용 시 보안·개인정보 우려는 제도적으로 실재하는 문제**
KISA는 생성형 AI 개발·활용 시 개인정보 처리에 관한 별도 안내서를 공개했습니다. 이 규제 환경이 ANG의 로컬 AI 서버 구조를 단순한 기능이 아닌 필수 설계 조건으로 만듭니다.
[출처: KISA 생성형 AI 개인정보 안내서](https://www.kisa.or.kr/2060301/form?lang_type=KO&page=1&postSeq=41)

**📌 KERIS · 교육 디지털 전환은 선택이 아닌 구조적 과제**
KERIS의 「교육 디지털 전환 기본계획 수립을 위한 기초 연구」는 학교 조직의 행정·문서·일정·정보관리 방식이 디지털 기반으로 개선될 필요가 있다고 명시합니다.
[출처: KERIS 교육 디지털 전환 기초 연구](https://www.keris.or.kr/main/ad/pblcte/selectPblcteRRInfo.do?mi=1138&pblcteSeq=13748)

## 08 — 자사 분석 (SWOT)

**기회를 잡고 약점을 관리한다**

|  | 내용 |
| --- | --- |
| **S 강점** | 조직 맞춤형 데이터 누적 구조 / 로컬 서버 기반 보안성 / 익숙한 칸반+캘린더 UI / 모듈화 설계로 도입 부담 최소화 |
| **W 약점** | 초기 문서 업로드 작업 필요 / 로컬 서버 하드웨어 비용 / 사용자 적응 기간 2~4주 / 신생 조직은 초기 AI 효용 낮음 |
| **O 기회** | 교육부 대학 DX 정책 지원 사업 연계 (B2G) / 1~2년 주기 담당자 교체로 인수인계 수요 상시 존재 / 디지털 전환 수요 급증 |
| **T 위협** | 노션·슬랙 등 기존 툴의 AI 기능 추가 가능성 / 대형 ERP 업체 AI 모듈 진입 / 대학 예산 부족으로 도입 결정 지연 |

**SWOT 대응 전략**

- 초기 데이터 부담 → 기존 파일 일괄 업로드 기능으로 진입 장벽 최소화
- 적응 기간 → 칸반+캘린더 UI로 별도 교육 없이 바로 사용 가능하게 설계
- 경쟁 툴 대비 → "우리 조직 데이터 기반" 포인트로 차별화. 범용 툴이 채우지 못하는 영역 공략
- 예산 부담 → 모듈별 선택 도입으로 소규모 예산 조직도 단계적 시작 가능

## 09 — 수익 모델

**어떻게 돈을 버는가**

부서 단위로 시작해 성과를 증명하고 점차 확장하는 구조입니다.

| 단계 | 형태 | 주요 내용 |
| --- | --- | --- |
| 1단계 · 초기 구축 | 부서 1개 기준 | 로컬 AI 서버 셋업, 기존 문서 OCR 변환, 초기 AI 학습 데이터 가공, UI 커스터마이징
( 스타트업이나 신생조직은 1차 타깃에서 무료배포 후 데이터 축적 ) |
| 2단계 · 유지보수 | 월 구독 · 부서 단위 | LLM 모델 업데이트, 서버 모니터링, 추가 데이터 학습, 기술 지원 (월 2회) |
| 3단계 · 확장 | 기능 단위 라이선스 | 문서 자동화 모듈, 캘린더 예측 모듈, 업무 보드 모듈, 타 부서 확장 도입 |

> 💰 수익 구조 핵심: 초기 구축비로 진입 후, 유지보수 구독으로 안정적 반복 수익 확보. 사용할수록 데이터가 쌓여 이탈 가능성이 낮고, 장기 고객 유지율이 높습니다.

## 10 — 개발 및 운영 전략

**단계별로 증명하고 확장한다**

**1단계 · 파일럿 운영 — 핵심 기능 3가지 검증**
문서 재활용 / 업무 보드 / 캘린더 알림 3개 모듈로 1개 부서에서 3개월 파일럿. 실제 업무 시간 단축 데이터를 측정해 도입 ROI 근거를 확보합니다.

**2단계 · 피드백 기반 개선 — 실사용 데이터로 완성도 상향**
파일럿 사용자 피드백으로 UI/UX 및 AI 정확도 개선. 성공 사례를 레퍼런스로 정리해 다음 도입 조직 설득 자료로 활용합니다.

**3단계 · 조직 내 확장 — 타 부서로 단계적 도입**
파일럿 성과를 바탕으로 대학 내 타 부서로 확장. 교육부 스마트 캠퍼스 지원 사업 연계를 통해 B2G 판로 개척.
